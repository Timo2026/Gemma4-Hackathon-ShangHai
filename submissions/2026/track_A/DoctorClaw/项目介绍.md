## 项目概述

**DoctorClaw**（代码仓库名 DocClaw）是一款面向临床医生的医疗 AI 工作台，参赛于 Gemma 4 Good Hackathon 上海站 Track A。项目以 **Gemma 4 31B** 大模型为核心引擎，帮助医生在门诊问诊、病历书写、检查解读与随访管理等场景中，获得安全、可控、可确认的智能化辅助。

传统医疗信息化系统侧重数据录入，而 DoctorClaw 聚焦「医生如何使用 AI」：将各类医疗能力封装为可启用、可自定义、可分享的 **Skill（技能）**，让 AI 真正融入医生的日常工作流，而非孤立对话工具。

## 核心亮点

- **双轨 AI 架构**：Skill 模式走结构化输出短路路径，保障病历质量；Agent 模式基于 DeepAgents + LangGraph + MCP，支持复杂多步推理与工具调用。
- **病历结构化**：问诊对话经单次 LLM 调用生成符合 JSON Schema 的结构化病历，配合 Pydantic 校验与 field_diff 事实比对，医生通过 HITL（人在回路）确认后落库。
- **技能生态**：个人技能管理、技能广场与 ClawHub 开放技能接入，支持科室级能力共享与复用。
- **患者连续管理**：患者队列、随访计划与通知中心，覆盖实时诊疗与计划性任务两类场景。

## 技术栈

前端采用 React + TypeScript + Vite；后端基于 FastAPI + SQLAlchemy；AI 层使用 Gemma 4 Structured Output 与 SSE 流式推送；Agent 层集成 MCP 工具服务与人机协同断点续跑能力。
