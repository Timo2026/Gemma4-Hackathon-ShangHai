## 一、模型选型背景

Union·由你系统采用多模型混合架构，支持云端API、本地llamacpp、Ollama等多种模型服务。在本地部署场景中，我们选择**Gemma 4**作为核心本地模型之一，基于以下技术考量：

### 1.1 业务需求约束

| 需求维度 | 具体要求 | Gemma 4适配性 |
|----------|----------|---------------|
| **离线运行** | 工厂车间无外网，必须100%本地推理 | ✅ 开源模型，可完全离线部署 |
| **中文支持** | 中文指令理解、技术术语处理 | ✅ 多语言预训练，中文表现优秀 |
| **硬件适配** | 工控机8-32GB内存，无GPU或低端GPU | ✅ 多规格可选(2B/7B/27B) |
| **响应速度** | 单次推理<5秒(8核CPU) | ✅ 2B/7B规格推理速度快 |
| **指令遵循** | 结构化参数提取、JSON输出 | ✅ 指令微调版本表现优异 |

### 1.2 Gemma 4 规格对比

| 规格 | 参数量 | 内存需求(Q4) | 推理速度(8核CPU) | 适用场景 |
|------|--------|-------------|------------------|----------|
| **Gemma 4 2B** | 2B | ~1.5GB | ~1-2秒 | 轻量级对话、快速响应 |
| **Gemma 4 7B** | 7B | ~4.5GB | ~3-5秒 | 主力模型、平衡性能 |
| **Gemma 4 27B** | 27B | ~16GB | ~10-15秒 | 高质量输出、复杂推理 |

**选择策略**：根据运行时环境自动选择最优规格：
- **内存≥24GB**: 优先使用27B规格(高质量)
- **内存8-24GB**: 使用7B规格(平衡)
- **内存<8GB**: 降级到2B规格或纯规则引擎

### 2.2 模型选择流程

```python
# 模型选择逻辑 (model_registry.py)
def select_with_fallback(task: str = "chat") -> list:
    """
    返回降级链: [cloud_best, local_api_best, local_config_best, ollama_best]
    """
    # 1. 获取所有可用模型(按质量评分排序)
    all_models = get_all_ranked()
    
    # 2. 按来源分类
    cloud = [m for m in all_models if m["source"] == "cloud"]
    local = [m for m in all_models if m["source"] == "local"]
    ollama = [m for m in all_models if m["source"] == "ollama"]
    
    # 3. 构建降级链
    chain = []
    if cloud: chain.append(cloud[0])  # 云端最优
    if local: chain.append(local[0])  # 本地次优(含Gemma 4)
    if ollama: chain.append(ollama[0])  # Ollama兜底
    
    return chain
```

## 三、质量评分机制

### 3.1 评分算法

```python
def _score_local_api_model(name: str) -> float:
    """本地API模型质量评分(启发式，零硬编码)"""
    score = 60  # 基础分
    
    # 1. 参数规模启发(越大越智能)
    param_match = re.search(r'(\d+\.?\d*)\s*[bB]', name)
    if param_match:
        param_b = float(param_match.group(1))
        score += min(param_b * 3, 30)  # 上限30分
    
    # 2. 中文模型加分(指令理解更好)
    if any(k in name_lower for k in ("qwen", "glm", "gemma", "baichuan", "yi")):
        score += 15
    
    # 3. 指令微调加分(遵循指令更好)
    if any(k in name_lower for k in ("chat", "instruct", "it", "-sft", "-dpo")):
        score += 10
    
    # 4. 量化模型适度降分(精度损失)
    if any(k in name_lower for k in ("q4", "q5", "q8", "int4", "int8", "mxfp4", "gguf")):
        score -= 5
    
    # 5. 大模型加分(更聪明)
    if any(k in name_lower for k in ("70b", "72b", "65b", "120b", "110b", "405b")):
        score += 10
    
    return max(min(score, 99), 30)
```

### 3.2 Gemma 4 评分示例

| 模型名称 | 基础分 | 参数加分 | 中文加分 | 指令加分 | 量化减分 | 总分 |
|----------|--------|----------|----------|----------|----------|------|
| `gemma2:2b` | 60 | +6 | +15 | +10 | -5 | **86** |
| `gemma2:7b` | 60 | +21 | +15 | +10 | -5 | **101→99** |
| `gemma2:27b` | 60 | +30 | +15 | +10 | -5 | **110→99** |
| `gemma2:7b-instruct` | 60 | +21 | +15 | +10 | -5 | **101→99** |

**说明**: 评分上限为99分，确保云端模型(通常90-95分)仍具竞争力。

## 四、硬件自适应策略

### 4.1 内存检测与模型选择

```python
# environment_detector.py
def _detect_memory(self) -> float:
    """检测系统内存(GB)"""
    try:
        page_size = os.sysconf("SC_PAGE_SIZE")
        total_pages = os.sysconf("SC_PHYS_PAGES")
        return round(page_size * total_pages / (1024**3), 1)
    except Exception:
        return 0.0

# 自适应选择逻辑(伪代码)
memory_gb = detect_memory()
if memory_gb >= 24:
    preferred_model = "gemma2:27b"  # 高质量
elif memory_gb >= 8:
    preferred_model = "gemma2:7b"   # 平衡
else:
    preferred_model = "gemma2:2b"   # 轻量
```

### 4.2 降级链与健康检查

```python
# 启动时健康检查
def _probe_model_health(model_config: dict) -> bool:
    """探测模型端点是否健康"""
    try:
        url = f"{model_config['api_url']}/models"
        req = urllib.request.Request(url)
        with urllib.request.urlopen(req, timeout=2) as resp:
            return resp.status == 200
    except Exception:
        return False

# 降级链过滤
fallback_chain = []
for model in raw_chain:
    if model["source"] == "cloud":
        fallback_chain.append(model)  # 云端模型总是保留
    elif _probe_model_health(model):
        fallback_chain.append(model)  # 本地模型需要健康检查
```

## 八、总结

选择Gemma 4作为Union·由你系统的核心本地模型，基于以下综合考量：

1. **开源许可**: 允许完全离线部署，满足工厂车间无外网要求
2. **多规格适配**: 2B/7B/27B三种规格覆盖不同硬件环境
3. **中文优化**: 多语言预训练，中文指令理解准确
4. **社区活跃**: Google官方维护，持续更新优化
5. **工具链成熟**: Ollama/vLLM/llama.cpp等推理引擎全面支持

通过与现有架构的深度集成，Gemma 4能够：
- **自动发现**: 端口1280等本地服务自动探测
- **智能降级**: 云端→本地→Ollama→规则引擎多层fallback
- **硬件自适应**: 根据内存自动选择最优规格
- **健康监控**: 启动时探测模型可用性，运行时动态切换

该设计确保系统在任何硬件环境下都能提供最优的AI能力，同时保持100%离线可用性。
