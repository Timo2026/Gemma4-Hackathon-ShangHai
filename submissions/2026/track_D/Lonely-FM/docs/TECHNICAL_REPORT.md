# Lonely FM 技术报告

PDF 版技术报告：`docs/Lonely-FM-Gemma4-Hackathon-Submission.pdf`  
演示视频：https://youtu.be/RzLeD7CA_Sw

## 1. 项目背景

Lonely FM 面向一个具体但普遍的社会问题：很多人并不是没有联系人，而是在低落、疲惫、深夜或独处时，不知道该把一句“我好累”说给谁听。我们希望用 Gemma 4 做一个更低门槛、更私密、更像通话的语音陪伴工具。

产品定位不是心理治疗，也不替代真实人际关系；它提供的是“先听完、再回应”的第一层陪伴。

## 2. 为什么选择 Gemma 4

我们选择 Gemma 4 的核心原因是它适合本地部署和云端备选并存：

- **端侧部署**：Lonely FM 的很多对话具有私密性，本地 Gemma 4 可以让用户把敏感内容尽量留在自己的设备中。
- **模型规格灵活**：项目默认使用 `gemma4:12b-mlx`，也兼容队友和测试者本地的 `gemma4:e4b`、`gemma4:21b` 等标签。
- **云端扩展**：没有本地模型的用户可以用自己的 Gemma 4 API key 继续体验，便于移动端和远程评审。
- **上下文理解**：Gemma 4 负责将文本、情绪信号、角色人设、长期记忆和安全边界综合为自然回应。

本项目提交重点展示 Gemma 4 的 **Edge / 本地部署能力**，而不是把大模型只作为一个普通云端聊天接口使用。

## 3. 系统架构

```text
用户浏览器
  ├─ 麦克风输入
  ├─ VAD / turn-taking
  ├─ 声线选择
  ├─ 登录与记忆控制
  └─ WebSocket
       ↓
FastAPI 实时后端
  ├─ STT 转写
  ├─ 情绪 / 语速 / 停顿分析
  ├─ Supabase 长期记忆读取
  ├─ Gemma 4 本地或云端生成
  ├─ TTS 合成
  └─ 音频与字幕流式返回
       ↓
前端播放队列
  ├─ 实时字幕
  ├─ 同心圆语音动效
  └─ 林屿 / 阿婉语音回复
```

## 4. 关键模块

### 4.1 Gemma 4 服务

文件：`backend/services/gemma.py`

该模块负责：

- 检测本地 Ollama 是否可用。
- 在多个 `gemma4:*` 标签中选择可用模型。
- 支持 OpenAI-compatible endpoint 和 Google AI API 作为云端路径。
- 在连接阶段预热模型，降低第一轮回应空白。
- 使用流式生成，把回复尽早交给 TTS。

### 4.2 实时语音管线

文件：`backend/routers/ws.py`

WebSocket 管线把一次语音对话拆成多个阶段：

1. 接收前端音频。
2. 判断用户是否真的说完。
3. 转写语音。
4. 识别情绪和对话意图。
5. 调用 Gemma 4 生成回复。
6. 将回复切成适合语音播报的短片段。
7. TTS 合成并回传音频。

为了减少“文字先出来很久才有声音”的问题，后端会优先产生短的第一语音片段，再逐步补足后续内容。

### 4.3 角色与安全提示词

文件：`backend/prompt/persona.py`

林屿和阿婉不是简单换声线，而是不同的陪伴风格：

- 林屿：低沉、安静、克制，像深夜电台，适合倾听和缓慢展开。
- 阿婉：温柔、轻松、治愈，适合日常陪伴和情绪承接。

提示词要求模型：

- 先回应用户刚刚说的具体内容。
- 少用套路安慰。
- 不用书面总结腔。
- 不在每次回复前机械地说“嗯”。
- 遇到危机内容时给出安全回应，并建议寻求现实帮助。

### 4.4 长期记忆

文件：`backend/services/supabase_memory.py`、`supabase/migrations/20260604_create_memories.sql`

登录用户的记忆保存到 Supabase，按用户身份隔离。访客模式只做临时会话，不写入长期记忆。用户可以删除已经保存的记忆。

## 5. 延迟优化

语音陪伴最容易失败的地方不是 UI，而是等待感。我们做了以下优化：

- **连接页预热**：用户点击角色后先进入“正在连接”，这段时间用于检测 Ollama、加载 Gemma 4 和准备首句问候。
- **keep-alive**：本地模型保持加载，减少后续轮次冷启动。
- **短首句策略**：首个语音片段控制更短，让声音尽快出现。
- **流式文本 / 音频队列**：Gemma 4 输出与 TTS 合成解耦，前端按队列播放。
- **禁用浏览器 TTS 混音**：避免同时出现系统男声和 MiniMax 女声的问题。

## 6. 数据合规

Track D 项目需要明确隐私保护，本项目采取以下策略：

- 本地 Gemma 4 优先，减少私密内容离开用户设备的机会。
- 云端 API key 是显式选项，不默认上传给第三方模型。
- 访客不保存长期记忆。
- 登录用户的记忆通过 Supabase Auth 绑定到用户账号。
- 记忆可删除，产品文案明确“记忆由你决定保留”。
- 安全边界提示模型不做医学诊断或专业治疗承诺。

## 7. 后续计划

- 接入更强的情绪语音模型，提升声线自然度和停顿感。
- 加强端侧 ASR 和更稳定的 VAD。
- 提供一键本地部署脚本，降低测试者安装门槛。
- 增加用户可视化记忆管理页面。
- 在移动端提供云端 Gemma 4 和本地网络代理两种模式。
