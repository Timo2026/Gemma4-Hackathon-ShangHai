# 端侧 / 边缘部署指南（Gemma 4 4B）

赛道 D 的隐私承诺核心：**默认的情绪陪伴对话与函数调用全部在端侧/本地完成，敏感数据不出设备。**

## 方案一：Ollama（最快，桌面/边缘盒子）

```bash
# 安装 Ollama 后
ollama pull gemma-4-4b-it-q4         # Q4_K_M 量化，约 3GB
ollama serve                          # 暴露 OpenAI 兼容端点 :11434/v1
```

后端 `GEMMA_EDGE_ENDPOINT=http://127.0.0.1:11434/v1` 即可对接，代码无需改动。

## 方案二：llama.cpp（更细的量化控制）

```bash
./llama-server -m gemma-4-4b-it-Q4_K_M.gguf --port 11434 -c 4096 --jinja
# --jinja 开启工具模板，启用原生函数调用
```

## 方案三：手机端（产品路线图）

- Android：通过 MediaPipe LLM Inference API 加载 Gemma 4 4B（int4）。
- 思路：把 `function_calling` 的编排循环移植到端上 runtime，工具实现（记录/评分/
  危机干预）本就为纯本地逻辑，天然适合端侧，云端仅在「多模态化验单解析」且用户
  授权时才介入。

## 选 4B 的理由

| 规格 | 端侧可行性 | 函数调用稳定性 | 共情对话质量 |
|------|-----------|--------------|------------|
| 2B   | 好         | 偏弱          | 一般        |
| 4B   | **好（量化后约 3-4GB）** | **稳定** | **好** |
| 26B MoE | 端侧不可行 | 很好        | 很好        |
| 31B Dense | 端侧不可行 | 很好      | 很好        |

4B 是「隐私优先的端侧」与「可靠函数调用 + 有温度对话」之间的甜点位。
